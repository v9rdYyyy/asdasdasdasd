# Family Bot v2

Отдельный проект без `.env`, с разнесением по файлам и slash-командами.

## Что исправлено

- больше нет зависимости от `.env`;
- больше нет обычных префиксных команд, значит не нужен `Message Content Intent`;
- есть отдельный `config.json`;
- есть глобальный бот-админ по ID, который может управлять ботом даже без Discord Administrator;
- проект разбит на модули: конфиг, база, команды, views, сервисы, embeds.

## Почему у тебя сейчас не запускается старый бот

По твоим логам главная ошибка такая:

`discord.errors.LoginFailure: Improper token has been passed.`

Это значит не ошибка логики бота, а неверный токен. Обычно причина одна из этих:

- вставлен не тот токен;
- вставлен токен с префиксом `Bot `;
- токен истёк или был пересоздан;
- в начале или в конце есть пробел;
- токен попал в публичный доступ и его нужно перевыпустить.

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

1. Открой `config.json`
2. Заполни:

```json
{
  "token": "ТВОЙ_ТОКЕН_БОТА",
  "global_bot_admin_ids": [123456789012345678],
  "db_path": "family_bot.sqlite3",
  "log_level": "INFO",
  "sync_commands_globally": true
}
```

### Что такое `global_bot_admin_ids`

Это список ID пользователей, которые считаются админами именно для этого бота.
Они смогут использовать команды бота даже если у них нет права `Administrator` на сервере.

## Запуск

```bash
python bot.py
```

## Команды

Все команды slash, без `!`.

### Первичная настройка

```text
/setup ids
```

Поля:
- `result_channel_id` — канал итогов заявок
- `voice_channel_id` — голосовой канал "Проверка"
- `review_role_id` — роль рекрутов
- `applications_category_id` — категория заявок, можно пусто
- `archive_category_id` — архивная категория, можно пусто
- `server_name` — название сервера в сообщении пользователю

### Показать настройки

```text
/setup show
```

### Развернуть панель

```text
/panel deploy
```

Можно указать `channel_id`, а можно просто вызвать в нужном канале.

### Открыть или закрыть набор

```text
/recruitment open
/recruitment close
```

### Управление бот-админами

```text
/botadmin add
/botadmin remove
/botadmin list
```

## Как работает логика

1. Публикуется одна закреплённая панель.
2. Пользователь нажимает кнопку.
3. Бот показывает кнопку `Заполнить анкету`.
4. Открывается форма из 5 обязательных полей.
5. После отправки создаётся приватный канал заявки.
6. Канал виден только роли рекрутов и боту.
7. Внизу канала кнопки:
   - `Одобрить`
   - `Отправить в резерв`
   - `Отказать`
8. После `Одобрить` или `Отправить в резерв` появляется блок `Как прошёл обзвон?`
9. После финального решения канал не удаляется, а скрывается и уходит в архив.

## Что отправляется

### Одобрить
- ЛС пользователю с текстом про заход в голосовой канал
- лог в канал итогов
- затем в канале заявки появляется вопрос про итог обзвона

### Отправить в резерв
- ЛС пользователю
- лог в канал итогов
- затем в канале заявки появляется вопрос про итог обзвона

### Отказать
- запрашивается причина через модальное окно
- сообщение уходит в ЛС и в канал итогов
- канал скрывается в архив

### Не принят после обзвона
- запрашивается причина
- сообщение уходит в ЛС и в канал итогов
- канал скрывается в архив

## Важное по Discord Portal

Для этого проекта достаточно стандартных настроек. Включать `Message Content Intent` не нужно.

## Если снова будет 401 Unauthorized

Проблема точно в токене, а не в коде.
Сделай новый токен в Discord Developer Portal и вставь его в `config.json`.
